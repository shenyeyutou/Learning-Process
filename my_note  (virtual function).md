###虚函数和消息的动态绑定

##1.消息的绑定

1>静态绑定

静态绑定（Static Binding）  
在编译期就决定调用哪个函数。默认情况下，C++ 的成员函数调用都是静态绑定。
所以在你的例子里：

```cpp

void func1(A& x)
{	......
	x.f(); //调用A::f还是B::f ？A::f
	......
}
void func2(A *p)
{	......
	p->f(); //调用A::f还是B::f ？A::f
	......
}

A a;
func1(a);   // 调用 A::f
func2(&a);  // 调用 A::f

B b;
func1(b);   // 调用 A::f
func2(&b);  // 调用 A::f
```

即使传入的是 B 类型对象，函数调用依然解析为 A::f，
因为编译器在编译时只知道参数类型是 A& 或 A*。

2>动态绑定

实际需求：需要在func1（或func2）中根据x（或p）
实际引用（或指向）的对象来决定是调用A::f还是B::f。

在C++中，在基类中用虚函数来实现动态绑定。

```cpp

class A
{		int x,y;
	public:
		virtual void f(); //虚函数
};
class B: public A
{		int z;
	public:
   		void f(); 
   		void g();
};
````

3>一个是在编译期，一个是在运行期

##2.虚函数

1>虚函数是指加了关键词virtual的成员函数。其格式为：

							virtual <成员函数声明>;

2>notice:(key points)

a.只有类的成员函数才可以是虚函数，但静态成员函数不能是虚函数。

b.构造函数不能是虚函数，析构函数可以（往往）是虚函数。
（原因：构造函数不能是虚函数，因为对象在构造时还没有建立虚函数表 (vtable)，编译器必须在编译期就知道对象的确切类型以分配内存和初始化成员。虚函数依赖运行时动态绑定，而构造过程本身需要静态确定性。）

c.只要在基类中说明了虚函数，在派生类、派生类的派生类、...中，
同型构的成员函数都是虚函数（virtual可以不写）。`

d.只有通过指针或引用访问对象类的虚函数时才进行动态绑定。
-------------------------------------------------

e.基类的构造函数和析构函数中对虚函数的调用不进行动态绑定。

e.g

```cpp
class A
{  ......
  public:
	A() { f(); }
	~A() { f(); }
	virtual void f();
	void g();
	void h() { f(); g(); }
};
class B: public A
{  .......
 public:
   B() { ...... }
	~B();
	void f(); 
	void g(); 
};

......
A a;  //调用A::A()和A::f
a.f();  //调用A::f
a.g();  //调用A::g
a.h();  //调用A::h、A::f和A::g
//a消亡时会调用A::~A()和A::f

B b;  //调用B::B()、A::A()和A::f
b.f();  //调用B::f
b.g();  //调用B::g
b.h();  //调用A::h、B::f和A::g
//b消亡时会调用B::~B()、A::~A()和A::f（上节课内容）
```
关注最后一个动态绑定（从A内部调用，却有一个this指针）

```cpp
a more complex example:

class A
{  ......
  public:
	A() { f(); }
	~A() { f(); }
	virtual void f();
	void g();
	void h() { f(); g(); }
};

class B: public A
{  .......
 public:
   B() { ...... }
	~B();
	void f(); 
	void g(); 
};

A *p;   //p是A类（基类）指针
p = &a; //p指向A类对象
p->f();  //调用A::f
p->g();  //调用A::g
p->h();  //调用A::h, A::f和A::g
p = &b;  //p指向B类对象
p->f();  //调用B::f
p->g();  //调用A::g，非虚函数采用静态绑定
p->h();  //调用A::h, B::f和A::g
p = new B;  //调用B::B(), A::A()和A::f
.......
delete p;  //？
```

#这里的 delete p; 是一个经典陷阱：

如果 基类析构函数不是虚函数，那么 delete p; 只会调用 A::~A()，不会调用 B::~B()。
→ 结果：派生类资源泄漏，析构不完整。

如果 基类析构函数是虚函数，那么 delete p; 会先调用 B::~B()，
再调用 A::~A()，保证完整析构。

因此，一旦类中有虚函数，就必须把析构函数也声明为 virtual。
这是 C++ 面向对象设计的黄金法则之一。(这也是对上述要点的诠释。)

ai:

🔹 总结规律:

虚函数调用 → 动态绑定，取决于对象实际类型。

非虚函数调用 → 静态绑定，取决于指针/引用的静态类型。

构造函数 / 析构函数中调用虚函数 → 退化为静态绑定（调用基类版本）。

析构函数必须声明为 virtual → 否则 delete p; 会导致资源泄漏。

#实现：虚函数表

🔹 vtable 的基本概念
vtable (Virtual Table)：一张由编译器生成的函数指针表。

vptr (Virtual Pointer)：每个对象内部隐藏的指针，指向所属类的 vtable。

作用：当调用虚函数时，程序通过 vptr 查找 vtable，找到实际要执行的函数地址，实现动态绑定。

🔹 工作流程
类定义阶段

如果类中有虚函数，编译器会为该类生成一张 vtable。

vtable 中的每一项对应一个虚函数，存放的是函数的地址。

对象构造阶段

每个对象会被赋予一个隐藏的 vptr，指向该类的 vtable。

如果对象是派生类，vptr 会指向派生类的 vtable。

函数调用阶段

当通过基类指针/引用调用虚函数时，程序会通过 vptr 查找 vtable，找到实际的函数地址并执行。

这样就能根据对象的动态类型调用正确的函数版本。

对象 p (类型 B)
   └── vptr → vtable(B)
                 └── f → B::f
